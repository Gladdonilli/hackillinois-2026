# Shared API Conventions

Every LARYNX endpoint follows these conventions. No exceptions. Consistency here means the frontend can rely on a stable API contract without surprise format changes.

---

## Response Envelope

Every JSON response uses this shape. Every one.

```json
{
  "success": true,
  "data": { ... },
  "error": null
}
```

On failure:

```json
{
  "success": false,
  "data": null,
  "error": {
    "code": "PROCESSING_FAILED",
    "message": "Model inference timed out after 300s"
  }
}
```

Never return a bare object. Never return an error string outside the envelope. The frontend checks `response.success` and branches, period.

## TypeScript Interfaces

```typescript
// shared/types/api.ts

interface ApiResponse<T> {
  success: boolean;
  data: T | null;
  error: ApiError | null;
}

interface ApiError {
  code: ErrorCode;
  message: string;
}

type ErrorCode =
  | 'UPLOAD_TOO_LARGE'
  | 'INVALID_FORMAT'
  | 'PROCESSING_FAILED'
  | 'MODEL_UNAVAILABLE'
  | 'RATE_LIMITED';

// SSE event payloads
interface SSEEvent<T = unknown> {
  type: string;
  data: T;
  timestamp: number; // Unix ms
}

// LARYNX-specific
interface AnalysisResult {
  verdict: 'genuine' | 'deepfake';
  confidence: number; // 0-1
  formants: FormantFrame[];
  ema?: EMAFrame[]; // present if AAI model available
  processingTimeMs: number;
}

interface FormantFrame {
  timeMs: number;
  f1: number;
  f2: number;
  f3: number;
  intensity: number;
}

interface EMAFrame {
  timeMs: number;
  sensors: Record<EMASensor, { x: number; y: number }>; // mm, midsagittal plane
  tongueVelocity: number; // cm/s
}

type EMASensor = 'UL' | 'LL' | 'JAW' | 'T1' | 'T2' | 'T3';

```

## Request Headers

| Header | Direction | Value | Notes |
|--------|-----------|-------|-------|
| `X-Request-ID` | Response | UUID v4 | Generated by CF Worker on every request. Echo it in logs. |
| `X-Processing-Time-Ms` | Response | integer | Wall-clock time of Modal inference, not total round-trip |
| `Content-Type` | Request | `multipart/form-data` or `application/json` | Multipart for file uploads, JSON for text prompts |

## File Upload

- Method: `POST` with `multipart/form-data`
- Field name: `file` (audio) or `prompt` (text)
- Max sizes:
  - Audio: **10 MB** (covers ~5 min WAV at 16kHz mono)
  - Text prompt: **1 KB** (roughly 250 words, more than enough)
- Accepted audio formats: `.wav`, `.mp3`, `.flac`, `.ogg`
- Reject everything else with `INVALID_FORMAT`

CF Worker validates size before forwarding to Modal. Don't waste GPU time on a 500MB upload.

## SSE (Server-Sent Events)

For processing progress. LARYNX streams status updates during inference.

```
event: progress
data: {"stage":"extracting_formants","percent":30,"message":"Analyzing frequency bands..."}

event: progress
data: {"stage":"running_ema","percent":70,"message":"Computing articulatory positions..."}

event: result
data: {"success":true,"data":{...},"error":null}

event: error
data: {"success":false,"data":null,"error":{"code":"PROCESSING_FAILED","message":"GPU OOM"}}
```

Rules:
- Event types: `progress`, `result`, `error`, `heartbeat`
- Heartbeat every 15s to keep connection alive: `event: heartbeat\ndata: {}\n\n`
- Always terminate with either `result` or `error`, never leave the stream hanging
- Client should timeout after 300s and show a retry prompt

## Error Codes

| Code | HTTP Status | When |
|------|-------------|------|
| `UPLOAD_TOO_LARGE` | 413 | File exceeds 10MB (audio) or 1KB (text) |
| `INVALID_FORMAT` | 400 | Unsupported file type, malformed JSON, missing required fields |
| `PROCESSING_FAILED` | 500 | Model threw an exception, OOM, or timed out |
| `MODEL_UNAVAILABLE` | 503 | Modal container is cold-starting or GPU not allocated |
| `RATE_LIMITED` | 429 | More than 5 requests/minute (prevent demo-day queue bombing) |

## CORS

```typescript
// CF Worker CORS headers
const corsHeaders = {
  'Access-Control-Allow-Origin': 'https://your-project.pages.dev', // CF Pages origin only
  'Access-Control-Allow-Methods': 'GET, POST, OPTIONS',
  'Access-Control-Allow-Headers': 'Content-Type',
  'Access-Control-Max-Age': '86400',
};
```

Replace `your-project.pages.dev` with your actual CF Pages domain. During local dev, also allow `http://localhost:5173` (Vite default).

## Auth

None. This is a hackathon. No user accounts, no API keys on the client, no JWTs. The CF Worker is the only thing between the internet and Modal. Rate limiting is the only protection, and it's just to prevent accidental self-DDoS during demo.
