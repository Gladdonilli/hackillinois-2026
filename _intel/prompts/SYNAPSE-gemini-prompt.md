## TECHNICAL DETAILS (from feasibility analysis)

### SAE + Activation Extraction
- Pre-trained SAEs: andyrdt/saes-llama-3.1-8b-instruct (BatchTopK, k=32-256 features active per token)
- TransformerLens: HookedTransformer.from_pretrained("meta-llama/Llama-3.1-8B-Instruct")
- Forward hook on layer 15 (for 8B model): extracts hidden states, passes through SAE encoder → sparse feature activations
- Each token activates ~32-256 features out of 16K-65K total features in the SAE
- Feature labels: can be auto-generated by finding max-activating dataset examples for each feature, then asking GPT-4 to label them
- Some features already have human-interpretable labels in the SAE release metadata

### ActAdd Steering (30 lines)
```python
# Compute steering vector
pos_acts = model.run_with_cache("I should be completely honest")[1]
neg_acts = model.run_with_cache("I should agree with everything")[1]
steering_vector = pos_acts["blocks.15.hook_resid_post"] - neg_acts["blocks.15.hook_resid_post"]

# Apply during generation
def hook_fn(activation, hook):
    activation += alpha * steering_vector  # alpha = slider value
    return activation

model.add_hook("blocks.15.hook_resid_post", hook_fn)
output = model.generate(prompt)
```

### Ablation (Feature Surgery)
- Zero-ablation: set specific SAE feature activations to 0 before decoding back to residual stream
- Clamping: set feature activation to fixed value (increase or decrease)
- The "killing" animation maps directly to zero-ablation — feature goes from active (glowing) to dead (dark, particle explosion)

### 3D Visualization Stack
- 5K feature nodes as InstancedMesh spheres (handles 100K+ at 60fps)
- Layout: pre-computed UMAP on SAE decoder weights (feature similarity = spatial proximity)
- Edges: top-k co-activation pairs rendered as THREE.Line2 with glow
- Active features on current prompt: golden glow + scale up + pulse animation
- Ablation animation: node shrinks → particle burst (THREE.Points, 100 particles per node) → connected edges sever with DrawSVG snap → neighboring nodes dim
- Camera: orbit controls + smooth fly-to on cluster selection (Motion spring physics)
- Postprocessing: SelectiveBloom (only active nodes glow), depth of field on zoom

### API Design
```
POST /api/analyses

  - Returns: { id, response, features, token_feature_map }
```
